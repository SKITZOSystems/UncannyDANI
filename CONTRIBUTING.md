# Contributing to UncannyDANI

## Core Principle
UncannyDANI is a single-channel, terminal-based narrative simulation. Contributions must preserve that constraint.

## Hard Constraints
- No backend services or external APIs
- No multi-screen UI expansion
- No multi-channel input systems
- No persistent cross-session memory systems
- No real psychological inference or diagnostic behavior
- No drift away from the terminal-first interaction model

## Architecture Integrity
- UI renders only
- Engines compute only
- Memory stores session state only
- Drift modifies perception only

## Allowed Evolution
- Boot sequence refinement
- Drift behavior tuning
- Prompt styling evolution
- Narrative pacing adjustments
- Visual and audio atmosphere improvements

## Forbidden Drift
- Turning the project into a chatbot system
- Adding conversation-memory AI
- Adding branching story logic
- Introducing real-world data inference
- Expanding into multi-agent simulation

## Rule Of Thumb
If a change increases complexity without increasing perceptual effect, it is not a valid contribution.
